display

A propriedade CSS display especifica o tipo de caixa de renderização usada por um elemento. No HTML, os valores padrões da propriedade display são feitas a partir do comportamento descrito nas especificações HTML ou da folha de estilo padrão do navegador/usuário. O valor padrão em XML é inline.

Além dos muitos tipos diferentes de exibição de caixa, o valor none permite desativar a exibição de um elemento; quando você usa none, todos os elementos descendentes também tem a sua exibição desativada. O documento é renderizado como se o elemento não existisse na árvore do documento.

Para entender melhor o posicionamento dos elementos crei algumas divs e dentro de cada div outras divos onde displays diferentes serão usados para melhor exemplificar. Para exemplificar foi usado dois elementos diferentes, um span e uma div. A div tem como padrão seu display block, já o span tem como padrão seu display inline.

Display block

Por padrão as div vêm com display block e o span com display inline. Um elemeto com display bloco oculpa toda linha e mesmo que ele seja pequeno ainda sim toda a linha será oculpada e o proximo elemento tendo ele display block ou inline oculpara a linha debaixo. Foi colocado dois span em sequência para mostrar que como eles possuem um display inline é permitido ficar mais de um elemento na mesma linha desde que ele caibam.

Display inline-block

O elemento gera uma caixa de elemento de bloco que fluirá com o conteúdo circundante, como se fosse uma única caixa embutida (se comportando como um elemento substituído). Neste tipo de display, mesmo elementos que sejam inline podem recever configurações diferentes das que os display permite.

Display none

Neste exemplo temos 4 elementos dentro desta div, duas span e duas div, porém eles não aparecem, pois o display none esta ativado. Mas o que exatamente este display faz? Ele desabilita a exibição do elemento (sem afetar o layout); todos os elementos filhos também tem sua exibição desabilitada. O documento é renderizado como se o elemento não existisse. O faz parte do html, mas é como se na renderização ele não existisse.
